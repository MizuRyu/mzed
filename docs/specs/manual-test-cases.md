# 手動テストケース集

リリース前に毎回実施する回帰テスト。TC-ID は永続識別子として変更しない。

> 自動テスト済みの項目は「自動化済み」欄に `tests/instance_integration.rs` 等を明記し、手動実施から外してよい。

---

## セキュリティ（SEC）

### SEC-01 IPC 非 .md ファイル拒否

| 項目 | 内容 |
|---|---|
| **前提** | mzed が起動済み |
| **手順** | 別ターミナルで `mzed /etc/hosts`（非 .md ファイル）を実行する |
| **期待結果** | 既存ウィンドウに変化なし。クラッシュしない |

### SEC-02 IPC ディレクトリ開く

| 項目 | 内容 |
|---|---|
| **前提** | mzed が起動済み |
| **手順** | `mzed ./some-dir`（ディレクトリ）を別ターミナルで実行する |
| **期待結果** | OpenDir としてルート切替が行われる（従来どおり） |

### SEC-03 ソケット権限 0600

| 項目 | 内容 |
|---|---|
| **前提** | mzed が起動済み |
| **手順** | `ls -l "$TMPDIR"mzed-*.sock` を実行する |
| **期待結果** | `srw-------`（0600）が表示される |
| **自動化済み** | `tests/instance_integration.rs` の `ソケット権限は0600に設定される` |

### SEC-04 非 .md 相対リンク無効化

| 項目 | 内容 |
|---|---|
| **前提** | `./page.html` 等の非 .md 相対リンクを含む Markdown ファイルを表示中 |
| **手順** | そのリンクをクリックする |
| **期待結果** | WebView が外部 HTML に遷移しない。何も起きないか、無反応 |

### SEC-05 アンカーリンク正常動作

| 項目 | 内容 |
|---|---|
| **前提** | 見出しへのアンカーリンク（`#section`）を含む Markdown を表示中 |
| **手順** | アンカーリンクをクリックする |
| **期待結果** | ページ内の該当見出しへスクロールする |

### SEC-06 .md 相対リンクとアプリ内遷移

| 項目 | 内容 |
|---|---|
| **前提** | `./other.md` への相対リンクを含む Markdown を表示中 |
